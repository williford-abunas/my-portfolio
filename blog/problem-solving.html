TECHNICAL BLOG

Problem Solving

1. That time I was blocked on a simple problem:

  • What was the problem?

    • I can see in the console.log that my code is working and giving what is similar to the expected results.

  • What problem-solving techniques did you use?

    • I tried to revise the code in different ways. And of course, none of them worked. In the end, I have no other choice but to ask for help. I tried the test but I was not reading it carefully so when I asked a peer he directed me to what the test says. Then, I realized that I shouldn’t have added a space somewhere.

  • How did you feel throughout the process?

    • There’s a point when I feel hopeless where everything I’m trying is not working. There’s an “Aha” moment when you find out that the solution is actually just simple.

  • What did you learn?

    • I learned to pay attention not only to the console.log but to the test as well. I also learned that asking for help from a peer makes you step out and see a different perspective that will help you find the missing piece.

2. That time when I elegantly solved a problem:

  • What was the problem?

    • I can’t write functions from scratch. Translating instructions to code is a challenge.

  • What problem-solving techniques did you use?

    1. Practice, practice, practice. Try doing things again and again while verbalizing what hapeens in the process.
    2. Console logging and running test to check.
    3. Treating google and youtube as my go to friends
    4. Asking for help when completely stuck

  • How did you feel throughout the process?

    • I can’t exactly say that this problem is elegantly solved but I’m just recognizing the improvement I made as I can now better understand how to write a function from given instructions.

  • What did you learn?

    • I learned that by constant practice you can learn difficult concepts and skills.

3. Reflect on how confident you feel using each of these problem-solving techniques/processes: (Very confident, confident, a bit confident, not confident)

Pseudocode - not confident Googling - very confident
Trying something - very confident Asking your peers for help - a bit confident
Reading error messages - not confident Asking coaches for help - a bit confident
Console.logging - confident Improving your process with reflection - a bit confident
Rubber ducky method - a bit confident

4. Reflect on a time you were reluctant to ask for help. Consider what made you reluctant to do so. What might you try differently next time?

I was reluctant to ask for help because I can see that most of my peers are moving on steadily with the course. I was hesitant that my question would be outdated. Next time, I need to fire any question right away and adjust my working hours to suit available mentors.
